I thought that a good first post would be to talk about how I made this site.
If you look at the [github repo](https://github.com/dtjandra888/dtjandra888.github.io), this project is only consists of a few files, and something similar would be a great first project for aspiring CS students.

The project is broken down into two parts, the Jinja HTML templates (and the static JS/CSS) and the python script that renders it.
The site is automatically deployed using Github Actions and Github Pages.

## HTML
Jinja is a Python templating library that helps maintain a consistent layout across pages. This site uses a `base.html` template to share common elements like the navbar and styling.
Notice the `{% block %}` segments in this simplified version. 

```html
<!DOCTYPE html>
<html lang="en">
<head>
    <title>{% block title %}{% endblock %}</title>
</head>
	<!-- Navigation -->
	<div class="main-nav">
		<div class="container">
            <!-- Nav items -->
		</div>
    </div>

    <div class="container">
        {% block content %} {% endblock %}
    </div>
</body>
</html>
```

When rendering the site, these blocks are replaced with whatever we define it as in the other files.
Using this concept, we can create other Jinja files that "extend" this `base.html` template to create different pages

Example: `post.html`
```html
{% extends "base.html" %}

{% block extra_head %}
<link rel="stylesheet" href="/static/css/blog.css" />
{% endblock %}

{% block title %}{{ title }}{% endblock %}

{% block content %}
<article class="post">
    <header class="post-header">
        <h1 class="post-title">{{ title }}</h1>
    </header>

    <section class="post-body">
        {{ content | safe }}
    </section>

    <footer class="post-footer">
        <p><a href="/blog.html">← Back to Blog</a></p>
    </footer>
</article>
{% endblock %}
```

## Python script and CI/CD
After making the templates, they are rendered using the `build.py` script.
Rendering the templates is quite simple and only takes a few lines of code.
Theres some other logic for copying the css/js files and creating the output directories, but I've put the snippet below to capture the important parts.

```python
from pathlib import Path

from jinja2 import Environment, FileSystemLoader

# Environment setup
template_dir = Path('templates')
output_dir = Path('dist')
env = Environment(loader=FileSystemLoader(template_dir))

# Render index.html
template = env.get_template('index.html')
html = template.render()
(output_dir / "index.html").write_text(html, encoding="utf-8")
```

As for CI/CD with Github pages, we just need to run our python script and upload the build artifacts before deploying.
Deploying is also quite simple since we can use one of the prebuilt Github actions (actions/deploy-pages).

```yaml
name: Build and Deploy to GitHub Pages

# only deploy when pushed to main
on:
  push:
    branches: [ main ]
  workflow_dispatch:

permissions:
  contents: read
  pages: write
  id-token: write

concurrency:
  group: "pages"
  cancel-in-progress: true

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout repository
        uses: actions/checkout@v4
        
      - name: Set up Python
        uses: actions/setup-python@v5
        with:
          python-version: '3.11'
        
      - name: Install dependencies
        run: |
          python -m pip install --upgrade pip
          pip install jinja2 markdown
        
        # Our script here!
      - name: Build static site
        run: python build.py

      - name: Upload artifact for GitHub Pages
        uses: actions/upload-pages-artifact@v3
        with:
          path: dist

  deploy:
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    runs-on: ubuntu-latest
    needs: build
    steps:
      - name: Deploy to GitHub Pages
        id: deployment
        uses: actions/deploy-pages@v4   # prebuilt action
```
